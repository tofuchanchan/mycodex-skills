# 组件规范

## 导航

### 左侧业务导航

- 白底，顶部放欧税通 Logo。
- 一级菜单包含图标 + 文本，行高约 44-48px。
- 当前菜单使用浅蓝背景、蓝色文字和图标。
- 有子菜单的模块使用展开箭头，子菜单缩进显示。
- 菜单可显示红色消息点或数量徽标，例如 VAT 信件、消息中心。

常见模块：购买指南、购买服务、续费服务、应用中心、工作台、合规空间、VAT+、EPR服务、授权代表、检测认证、集团业务、消息中心、账户中心。

### 顶部快捷入口

- 右侧排列购物车、邀请有礼、优惠专区、帮助、头像/账号。
- 图标优先，文字简短。
- 当前促销入口可用红橙色强调，但不要抢过页面主操作。

## 按钮

| 类型 | 样式 | 用途 |
| --- | --- | --- |
| 主按钮 | 蓝底白字，约 32-44px 高，圆角 4-6px | 登录、购买服务、立即购买、发起申报 |
| 次按钮 | 白底灰边，深色文字 | 搜索、重置、导出、开票 |
| 文本按钮 | 蓝色文字 | 订单详情、发票信息、下载证书、返回上一级 |
| 危险文本 | 红色文字 | 驳回原因、终止服务、删除/取消类动作 |
| 禁用按钮 | 浅灰底/浅灰文字 | 售罄、不可操作、未满足条件 |

按钮文案必须是业务动作，不要只有“确定”“提交”“操作”这种废话按钮。

## 表单与筛选

- 输入框高度约 32px，白底，浅灰边框，圆角 4px。
- 占位符使用 `#999999`。
- 筛选区放在内容卡片顶部，常见字段：时间、订单号/服务单号、国家/地区、税号/公司名、服务到期日。
- 搜索和重置按钮靠近筛选字段。
- 复杂筛选不要默认堆太多，商户端不是让用户做数据库管理员。

## Tabs

- 列表页 tabs 使用边框式或卡片式 tab。
- 当前 tab 使用蓝色文字、蓝色边框或蓝色底部线。
- tab 文案可带数量，例如“已授权 (144)”“待授权 (160)”。
- 流程页主步骤不使用普通 tab，应使用步骤条。

## 表格

- 表头浅灰底或白底加粗，文字 `#333333`。
- 表格线使用 `#E8E8E8`。
- 操作列通常固定在右侧，操作使用蓝色文本链接。
- 分页放在右下或表格底部，显示总数、页码、每页条数。
- 表格适用于税号、申报、订单等列表，但必须提供清晰 tabs 和筛选。

表格列建议：

- 订单列表：服务名称、规格、数量、金额、订单状态、操作。
- VAT税号：税务类型、公司名称、VAT税号、本土税号、国家/地区、税率、申报周期、操作。
- VAT待申报：国家/地区、税号、公司名、申报类型、待申报期间、操作。

## 商品卡片

商品/服务购买页使用网格卡片：

- 卡片白底，圆角约 6-8px，弱阴影。
- 顶部展示服务缩略图、国旗或品牌水印。
- 中部展示服务名称，支持两行截断。
- 价格使用红色，原价可灰色删除线。
- 底部左右两个操作：“加入购物车”和“立即购买”。
- 售罄使用灰色遮罩和“已售罄”标记，操作禁用。

## 信息卡与详情表格

详情页常用白色卡片分区：

- 区块标题 16px 加粗，如“订单信息”“支付信息”“基本信息”“VAT申报信息”。
- 信息使用双列或多列表格展示，字段名浅灰/居右，字段值深色。
- 重要金额、应缴 VAT、优惠金额使用红色。
- 协议勾选、资料下载、预览等动作放在相关信息附近。

## 步骤条

VAT 申报详情页使用横向步骤条：

- 步骤为蓝色长条，已完成显示勾选图标。
- 当前步骤使用更浅/更亮蓝色或底部三角指示。
- 步骤文案必须描述业务阶段，例如“确认申报信息”“请上传销售数据”“验证已有税号”“查看算税结果”“添加其他申报项”“查看申报明细”。
- 步骤下方应有当前申报国家、公司名称、VAT税号、本土税号、应缴金额等摘要。

## 标签与状态

- 蓝色标签：待续费、待申报、进行中。
- 绿色标签：已授权、已完成、通过。
- 红色标签：驳回、欠费、异常。
- 灰色标签：未激活、不可用、已取消。
- 数量徽标使用红色小圆点或胶囊。

状态标签不要只有颜色，必须有文字。只靠颜色传达状态，属于给用户出视觉谜题。

## 加载骨架

- 骨架使用浅灰块模拟卡片、表格、输入框和图片区域。
- 骨架应保持最终布局尺寸，避免加载完成后大幅跳动。
- 工作台和购买页可使用骨架屏。

## 浮动工具

右侧浮动工具可包含：

- 邀请有礼
- 购物车
- 客服
- 公众号

图标使用圆形或小卡片，文字垂直排列，固定在内容右侧。不要遮挡表格操作列和分页。

## 异常页

404 页面结构：

- 插画居中偏左。
- 右侧或旁边显示标题：“诶呀！没有找到您要访问的页面！”
- 辅助文案：“请检查输入的参数是否正确”
- 主按钮：“返回首页”
- 测试环境可以保留底部沙箱警示条。
