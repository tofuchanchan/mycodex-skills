# OST商户端（User）系统设计规范

## 定位

本技能用于欧税通商户端/用户端 PC Web。页面服务对象是购买和使用欧税通服务的商户用户，典型任务包括登录、查看工作台、购买服务、管理订单、查看 VAT 税号、处理 VAT 申报、查看申报详情和处理异常页面。

商户端不是内部管理后台。它可以有表格和侧边栏，但必须让用户知道“我在哪、我要做什么、下一步点哪里、服务状态怎么样”。别把用户扔进后台表格迷宫里自生自灭。

## 使用流程

设计、生成或评审商户端页面时，按需读取：

1. `references/source-observations.md`：本规范的页面来源和实际观察结论。
2. `references/visual-style.md`：色彩、字体、间距、卡片、布局骨架。
3. `references/components.md`：按钮、表单、标签、表格、商品卡、步骤条、浮动工具。
4. `references/page-patterns.md`：登录、工作台、订单、购买、VAT、申报、404 等页面模式。
5. `references/interactions-copywriting.md`：交互状态、加载、错误、提示、文案语气。

## 全局设计原则

- **商户自助优先**：每个页面都要帮助用户完成服务购买、资料查看、申报处理、订单追踪等明确任务。
- **左侧导航稳定**：PC 端沿用左侧业务导航 + 顶部快捷入口 + 内容卡片区的整体框架。
- **蓝色主路径**：主操作、当前选中、链接、步骤完成态使用品牌蓝，避免一页几十种强调色开染坊。
- **白底卡片承载内容**：主内容以白色卡片、浅灰背景、轻阴影分区，保持清爽但不装成营销落地页。
- **表格可用但别后台化**：列表页可使用表格，但操作文案、状态标签、筛选条件必须面向商户可理解。
- **购买页偏商品陈列**：服务购买使用商品卡片、国家/类型筛选、价格展示、加入购物车/立即购买双操作。
- **流程页强引导**：VAT 申报详情等长流程必须用步骤条、当前步骤高亮、上下文摘要和结果区组织信息。
- **异常页友好明确**：404 等异常页使用插画 + 清晰原因 + 返回动作，不要甩一屏技术错误码。

## 快速视觉锚点

- 主色：`#0077FF`
- 页面背景：`#F5F7FA` / `#F7F8FA`
- 内容卡片：`#FFFFFF`
- 主文本：`#333333`
- 次文本：`#666666`
- 辅助文本：`#999999`
- 边框/分割：`#E7E7E7` / `#E8E8E8`
- 危险/价格：`#F5222D` / `#FF0000`
- 顶部和侧边栏以白色为主，当前菜单使用浅蓝底 + 蓝色文字/图标。

## 输出要求

当输出原型、Figma 设计或前端页面时：

- 明确使用商户端框架，不要套后台深色侧栏和运营后台视觉。
- 页面第一屏必须包含当前任务入口或当前状态，不要写一堆“欢迎使用”的废话文学。
- 操作文案使用用户能懂的业务词：如“购买服务”“订单详情”“发起申报”“下载证书”“返回首页”。
- 对涉及金额、税号、申报、证书、资料上传的内容，必须突出状态、下一步和风险提示。
- 如果用户要求移动端，需要先说明本规范来源是 PC Web，再做移动端适配推导。
