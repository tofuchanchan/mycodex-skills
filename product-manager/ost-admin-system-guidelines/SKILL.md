# OST后台系统设计规范

## 定位

本技能用于欧税通内部后台系统。后台页面服务对象是运营、客服、财务、交付、管理员等内部角色，核心目标是高效处理数据、快速筛选定位、批量操作和稳定管理。

如果页面面向商户用户、买家、客户自助操作、购买服务、用户中心、VAT申报自助流程，请改用 `ost-user-system-guidelines`。别把商户端画成内部后台，用户不是来考后台操作证的。

## 快速使用

创建或评审 OST 后台页面时：

1. 优先读取 `references/color-scheme.md` 获取后台色彩规范。
2. 读取 `references/components.md` 获取表格、表单、弹窗、按钮等后台组件规范。
3. 读取 `references/layout.md` 获取左侧菜单、顶部栏、内容区、筛选区和表格区布局规则。
4. 读取 `references/interactions.md` 获取筛选、校验、弹窗和状态反馈规则。
5. 读取 `references/typography.md` 获取字号和排版层级。

## 后台与商户端的边界

- 后台优先信息密度、批量处理、可审计、状态可追踪。
- 商户端优先用户理解、购买转化、流程引导、服务信任感。
- 后台可以使用高密度表格、复杂筛选、多列状态和批量操作。
- 商户端不要默认复刻后台表格地狱，除非用户明确要求做高密度管理页。

## 后台页面骨架

- 左侧深色或强管理感导航，支持多级菜单。
- 顶部全局导航包含搜索、语言/平台、刷新、账户操作。
- 内容区以筛选表单 + 数据表格 + 批量操作 + 分页为主要结构。
- 弹窗用于新增、编辑、确认、审核、分配等内部操作。
- 状态信息必须清晰可扫读，错误和风险操作必须可追溯。

## 输出要求

当使用本技能输出页面、原型或 UI 代码时，应明确符合“后台管理系统”气质：克制、稳定、高效、表格友好、操作路径清晰。若用户实际描述的是商户端或 `uat-user.evatmaster.com` 页面，应切换到 `ost-user-system-guidelines`。
