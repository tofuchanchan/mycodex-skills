# 交互与文案

## 交互状态

### 加载

- 工作台、购买页、列表页加载时使用骨架屏。
- 骨架屏应模拟真实布局，避免加载完成后布局跳动。
- 关键操作按钮加载中应禁用并显示进行中状态。

### 空状态

空状态应包含：

- 当前没有什么数据。
- 用户为什么会看到空状态。
- 下一步可以做什么。

示例：

- “暂无订单，您可以先购买服务。”
- “暂无待申报数据，请确认税号是否已授权或申报周期是否已开始。”

### 错误

- 错误文案使用用户能理解的业务语言。
- 不要把接口字段、异常类名、后端日志暴露给商户。
- 可恢复错误必须给出重试、返回、联系客服等动作。

示例：

- 好：`申报数据暂时加载失败，请稍后重试。`
- 差：`declareId invalid exception`。这玩意儿给用户看，等于让用户帮研发背锅。

### 危险操作

终止服务、取消订单、删除资料、跳过校验等操作必须：

- 使用明确动作文案。
- 展示影响范围。
- 必要时二次确认。
- 不要把危险操作和普通蓝色链接混在一起。

## 文案语气

- 使用清晰、直接、服务型中文。
- 面向商户用户，不要使用内部后台黑话。
- 按钮文案使用动词 + 对象，例如“购买服务”“发起申报”“下载证书”“返回首页”。
- 状态文案短而明确，例如“已授权”“待续费”“待申报”“已完成”“驳回”。

## 常用文案模式

### 面包屑

- `账户中心 > 我的订单`
- `VAT+ > VAT税号`
- `VAT+ > VAT申报 > 发起申报`

### 列表操作

- 搜索
- 重置
- 导出数据
- 导出订单明细
- 购买服务
- 发起批量申报
- 查询汇总年报

### 行操作

- 订单详情
- 发票信息
- 税号资料
- 绑定方法
- 下载证书
- 终止服务
- 跳过校验
- 发起申报
- 驳回原因
- 续费

### 帮助入口

- 帮助
- 客服
- 申报操作手册
- 观看教程

## 金额与税务信息

- 金额使用红色强调，保留货币符号。
- 折扣、优惠、实付款应成组展示。
- 税率、申报周期、应缴 VAT 等关键信息应靠近业务上下文。
- 德文/英文税务条目可以保留原文，但中文解释不能缺席。

## 可访问性与可扫读

- 链接和按钮不要只靠颜色区分，关键状态要有文字。
- 表格操作列不要过窄，避免操作文字挤成一坨。
- 商品卡片中的服务名两行截断，价格和操作按钮固定在底部。
- 浮动工具不得遮挡主要操作列、分页和金额合计。

## 禁止事项

- 不要把商户端做成纯后台：满屏筛选 + 表格 + 操作列，没有引导，用户看了想报警。
- 不要在购买页使用后台表格卖服务，服务商品需要卡片和价格感知。
- 不要滥用红色。价格和风险可以红，普通提醒别红成火灾现场。
- 不要把 UAT 黄色警示条当正式视觉资产。
- 不要用“操作成功”“请求失败”这种泛文案糊弄关键业务流程。
