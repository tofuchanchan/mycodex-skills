# 来源页面与观察结论

## 抓取范围

本规范依据欧税通 UAT 商户端 PC Web 页面整理，页面域名为 `uat-user.evatmaster.com`。抓取时只做 URL 直达、截图和 DOM/样式读取，不点击业务按钮。

覆盖页面：

- 登录页：`/user/login`
- 首页/工作台：`/workPlace`
- 订单列表：`/payCenter/orderManageIndex`
- 订单详情：`/payCenter/orderManageIndex/orderManageDetail`
- 购买服务：`/buy`
- VAT 税号列表：`/VAT/orderIndex`
- VAT 税号详情：`/VAT/orderIndex/VATPanorama`
- VAT 待申报列表：`/VAT/declareIndex`
- 申报详情：`/VAT/declareIndex/calculateTaxModalRevision-v2`
- 404：`/404`

说明：用户提供的“商品/服务管理”和“合规空间页”URL 均为 `/buy`。因此本规范把 `/buy` 作为购买服务/商品陈列页处理；如未来存在独立合规空间 URL，应补充截图后再细化。

## 视觉事实

- 页面使用 PC Web 固定框架：左侧导航、顶部快捷入口、主内容卡片区、右侧浮动工具。
- 主色为高饱和蓝：`#0077FF`，用于主按钮、选中态、链接、步骤条。
- 主背景为浅灰，内容容器为白色卡片，圆角约 8px，弱阴影。
- 字体主要为 `PingFang SC`，正文常用 14px。
- 表格页信息密度较高，但商户端仍通过面包屑、tabs、状态标签、操作链接降低迷路感。
- 商品购买页使用卡片网格，突出国家、服务名称、价格、加入购物车和立即购买。
- 申报详情页使用顶部横向步骤条组织流程，当前步骤用更强蓝色和指示三角强调。
- 404 使用插画 + 轻口语标题 + 返回首页按钮，明显区别于后台式错误页。

## 环境特有元素

- 页面底部存在黄色沙箱环境警示条：`当前是欧税通沙箱环境，仅供内部使用，禁止外传！`
- 该警示条属于 UAT 环境标识，不应进入正式商户端视觉规范，除非明确设计测试环境页面。
- 截图里可能出现测试账号、订单号、税号、公司名等数据；规范中不得固化这些具体业务数据。

## 与后台规范的区别

- 后台规范关注内部效率、批量处理、高密度表格。
- 商户端规范关注商户理解、服务购买、申报流程、状态透明、下一步行动。
- 商户端可以有表格，但要控制筛选复杂度，操作链接必须业务化。
- 商户端需要更多空状态、加载态、异常页、帮助入口和客服入口。
