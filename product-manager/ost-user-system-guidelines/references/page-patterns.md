# 页面模式

## 登录页

布局：

- 左右分栏，左侧为品牌 Logo、促销图、版权信息，右侧为登录表单。
- 右侧标题使用“欧税通云服务平台”，蓝色大字。
- 登录方式使用分段切换：账号登录 / 扫码登录。
- 登录子方式使用 tab：密码登录 / 验证码登录。
- 输入框宽度约 400px，高度约 44px。
- 登录按钮为整行蓝色主按钮。
- 底部提供“忘记密码”“立即注册”。

规则：

- 登录页可以更偏营销，但表单区必须干净、可信。
- 左侧促销图不要影响登录表单阅读。
- 错误提示放在字段附近或按钮上方，使用红色明确原因。

## 工作台

布局：

- 左侧导航 + 顶部快捷入口。
- 主内容由多个白色卡片组成。
- 可包含服务概览、订单概览、信件/工单/资产提醒、常用入口、优惠信息。
- 加载时使用骨架屏保持结构。

规则：

- 第一屏应优先展示商户当前待处理事项和服务状态。
- 数字、待办、异常状态要明显，不要藏在装饰卡片里。

## 订单列表

结构：

- 面包屑：账户中心 > 我的订单。
- 筛选区：下单时间、订单号/服务单号、搜索、重置。
- 右上操作：导出订单明细、开发票、购买服务。
- 状态 tabs：全部、待付款、支付待确认、已取消、已付款、已完成、已退款。
- 订单按时间分组，每个订单块包含服务信息、规格、数量、金额、状态、操作。

规则：

- 订单号、时间、金额、状态必须在一屏内可扫读。
- 操作链接使用蓝色，例如“订单详情”“发票信息”“再来一单”。
- 金额使用红色突出，原价可灰色删除线。

## 订单详情

结构：

- 面包屑 + 返回上一级。
- 信息卡：订单信息、支付信息。
- 商品表格：服务名称、规格、单价、数量、优惠分摊、小计、操作。
- 右下合计区：商品总计、优惠、实付款。
- 协议勾选区靠近支付/购买相关动作。

规则：

- 详情页要强调订单状态、支付状态、发票状态和实付款。
- 金额统一右对齐或成组展示。
- 订单信息和支付信息使用字段表格，避免散乱文字。

## 购买服务页

结构：

- 顶部类型导航：全部类型、VAT+、EPR服务、授权代表、检测认证、碳合规、优惠套餐、商标专利、境外工商等。
- 国家/地区筛选横向排列。
- 右侧搜索框按服务名称搜索。
- 商品服务按国家分组，使用卡片网格展示。
- 卡片底部提供加入购物车和立即购买。

规则：

- 购买页是商户端最接近交易转化的页面，卡片要比表格更适合。
- 国旗、国家、服务名、价格、促销信息必须清晰。
- 售罄状态用遮罩和禁用按钮，不要让用户误点。

## VAT税号列表

结构：

- 面包屑：VAT+ > VAT税号。
- 筛选区：税务类型、国家/地区、服务到期日、VAT税号/公司名称。
- 状态 tabs：已授权、待授权、未付费、已过期、已终止。
- 表格列：税务类型、公司名称、VAT税号、本土税号、国家/地区、税率、VAT申报周期、操作。
- 操作：税号资料、绑定方法、下载证书、终止服务。

规则：

- 税号类页面是强业务工具页，可以保留高密度表格。
- 操作链接必须保持横向可扫读，不要折成“更多”黑盒。
- 到期、终止、未付费等风险状态要明显。

## VAT税号详情

结构：

- 顶部摘要：VAT税号、状态、服务到期日、可申报期间。
- tabs：基本资料、申报、缴税、服务单、订单、信件、工单。
- 主区块：基本信息、VAT申报信息、B2B申报信息、资料信息、备注。
- 右侧时间线：下号完成、提交资料、购买等关键节点。

规则：

- 详情页左侧放结构化数据，右侧放进度时间线。
- 资料文件提供预览/下载动作。
- 备注、资料、税率日期等要清楚归组。

## VAT待申报列表

结构：

- 面包屑：VAT+ > VAT申报。
- 顶部 tabs：待申报、申报记录、在线缴税。
- 操作入口：申报操作手册、导出数据、发起批量申报、批量申报记录、查询汇总年报。
- 筛选区：国家/地区、申报周期结束月份、税号/公司名。
- 表格列：待申报国家/地区、税号、公司名、申报类型、待申报期间、操作。

规则：

- 待申报页要突出“发起申报”和“跳过校验”等当前动作。
- 对驳回、续费、补申报等状态使用标签或明确链接。
- 批量操作放右上，但不要压过单行主操作。

## 申报详情

结构：

- 面包屑 + 返回上一级。
- 顶部横向步骤条。
- 当前申报摘要：国家、公司名称、VAT税号、本土税号、应缴金额。
- 主区显示当前步骤对应内容，例如申报明细表、算税结果、上传销售数据等。
- 可有右下角教程视频浮层。

规则：

- 步骤条是核心导航，必须保留。
- 当前步骤必须有明显指示，不要只靠颜色差异。
- 表格中的德文/英文税务条目可保留原文，但应提供中文解释或并列表述。
- 应缴金额、税率、申报周期必须显眼。

## 404

结构：

- 白底页面，中心区域显示插画。
- 文案轻口语但明确：“诶呀！没有找到您要访问的页面！”
- 说明文字：“请检查输入的参数是否正确”
- 蓝色主按钮：“返回首页”

规则：

- 404 不显示侧边导航，保持轻量。
- 不要展示堆栈、路由名、接口错误。
- 可以保留品牌插画风格。
